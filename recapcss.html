Recap, CSS
Why exactly do we have to put a dot in front of a class name?

In CSS, the dot (.) before a class name tells the browser that you're selecting an element based on its class attribute. For example, .example targets all elements with the class example.
Why sometimes there's no dot (like when we use * or body)?

The asterisk (*) is a universal selector that selects all elements, while body selects the <body> tag specifically. We don't use a dot here because these are not classes, they're direct element selectors.
Can we combine multiple classes in one <div>?

Yes, you can apply multiple classes to one element by listing them in the class attribute separated by spaces, like this: <div class="class1 class2"></div>. Each class will apply its respective styles to the element.
Why should we use CSS styles instead of inline styles?

CSS styles in a separate file or style block keep your code cleaner and easier to maintain. Inline styles mix content with presentation, which can make updates difficult, especially on large websites. By using external or embedded CSS, you ensure consistency across multiple elements and pages.
Why doesn't my CSS work?

This usually comes down to simple syntax issues. One common mistake is forgetting to put a semicolon at the end of each CSS declaration. The semicolon acts as a separator, letting the browser know you've finished specifying a property. For instance, the following rule might not work as expected because it's missing a semicolon after the color property:

.example {   color: blue   font-size: 20px; }

To fix it, just add the missing semicolon:

.example {   color: blue;   font-size: 20px; }

Another common issue is forgetting to close the curly brackets {} around your CSS rules. If the browser doesn't see that closing bracket, it won't apply the styles at all. Here's an example where the rule is missing a closing bracket:

.example {   background-color: yellow;   font-weight: bold;

The corrected version should look like this:

.example {   background-color: yellow;   font-weight: bold; }

So always double-check your CSS for missing semicolons and ensure all curly brackets are properly closed!
Help! I created shapes, but they're not showing up on my website. What could be wrong?

Let's say you created some shapes in CSS like this:

.square {   width: 100px;   height: 100px;   background-color: lightgray; }
.circle {   width: 100px;   height: 100px;   border-radius: 50%; }

You might wonder, "Why isn't the circle showing up on my webpage?"

A common issue is forgetting to give the shape a background color. For example, in the circle class above, we defined the width, height, and border-radius to make it round, but since we didnâ€™t add a background-color, it remains invisible.

Here's the corrected CSS for the circle:

.circle {   width: 100px;   height: 100px;   border-radius: 50%;   background-color: yellow; }

After defining the width and height, always remember to "fill" your shapes by adding a background color (or any other styling like borders) to make them visible!
