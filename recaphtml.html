Recap, HTML
Why should the main page be called index.html?

The file index.html is the default file that web servers look for when loading a website. It acts as the entry point of your site. If you name it something else, like home.html, the server might not automatically find it, requiring users to type the full URL to access the page.
Why is proper syntax important?

Proper syntax ensures that your HTML is interpreted correctly by browsers. Incorrect syntax can lead to unexpected behavior or a broken webpage. For example, consider the following incorrect HTML code:

<p>Hello, world!<p>Welcome to my site!</p>

In this example:
- The first <p> tag is not properly closed before the second <p> starts.
- This mistake could cause all subsequent content to be incorrectly formatted, as the browser tries to interpret the missing closing tag.

The correct syntax should be:

<p>Hello, world!</p><p>Welcome to my site!</p>

By using proper syntax, you ensure your webpage is displayed correctly and consistently across different browsers.
Why is closing tags important?

Closing tags like </b> indicate where formatting should end. If you don't close a tag, the browser might apply formatting to more content than intended, causing layout issues and unexpected results.
Why does text not display next to each other?

You might think that using elements like <h1>Big Title</h1><h2>Medium Title</h2> would make the titles appear next to each other, like in a line. But that's not how it works! These elements are "block-level," which means they automatically take up the full width and start on a new line.

For example, if you write this HTML:

<h1>Big Title</h1><h2>Medium Title</h2><h3>Small Title</h3>

It won't show the titles in a row. Instead, it will look like this:

Big Title

Medium Title

Small Title


The same thing happens with paragraph tags like <p>Hello, World!</p><p>Welcome to my page!</p>. You might expect "Hello, World! Welcome to my page!" to be on one line, but it will actually display like this:

Hello, world!
Welcome to my page!

This happens because both headings and paragraphs are block elements, meaning they automatically start on a new line. If you want them to appear side-by-side, you can use inline elements like <span>, which do not force a new line.

On a pure HTML site, you technically don't need <p> tags or <span> tags if you just want text to flow together on a single line. You can write text directly in the HTML like this:

Hello, world! Welcome to my site!

And it will display inline, one after the other.
Why are some tags self-closing while others require a closing tag?

Self-closing tags, such as <br>, <img>, and <hr>, do not need a closing tag because they represent elements that do not contain any content or child elements. These tags perform specific functions, like inserting a line break or displaying an image, without enclosing any content.

In contrast, tags that require a closing tag, such as <b>text</b> or <p>paragraph content</p>, are used to wrap content or child elements. The opening tag marks the beginning of the element, while the closing tag signifies its end. This structure is essential for properly formatting or displaying the enclosed content on the webpage.
